# Security Policy

## Supported versions

Security updates are provided for the latest release line.

## Reporting a vulnerability

Please do not open public GitHub issues for security vulnerabilities.

Report privately with:
- affected package(s)
- affected version(s)
- reproduction details
- potential impact

Until a dedicated security contact address is configured, coordinate disclosure through repository maintainers in a private channel.

## Disclosure process
- Acknowledge receipt.
- Triage and reproduce.
- Prepare fix and tests.
- Publish patched release and advisory notes.
